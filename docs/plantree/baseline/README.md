# Baseline

Date: 2026-05-25

## Purpose

This folder keeps lightweight cross-project context for plans. It is not a full
architecture specification; authoritative behavior still lives in the contract
documents under `docs/`.

## Files

- [module-map.md](module-map.md) lists user-facing files and relevant contract
  anchors.
- [runtime-flows.md](runtime-flows.md) summarizes the high-level CCB user
  flows that plans may reference.
- [storage-and-state.md](storage-and-state.md) summarizes planning-relevant
  state and asset locations.
- [test-and-release-gates.md](test-and-release-gates.md) records likely
  verification gates for documentation and release-facing work.
- [risk-hotspots.md](risk-hotspots.md) records known planning risks.
